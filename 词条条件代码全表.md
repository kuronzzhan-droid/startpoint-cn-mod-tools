## §2 五种表布局(头部列 + 块基址)

**ability**(共 126 列):

- `c0` string_id 文本键(词条描述引用)
- `c1` unisonable true/false(false=仅主位Ⓜ)
- `c2` 词条类别串(attack_common 等 20 种,决定图标/大类)
- `c3` awake_kind 0=无 1=Switch觉醒替换 2=Addition觉醒追加
- `c4` awake_level 觉醒等级门槛
  (实测:Switch 行成对出现,`c3=1,c4=0` 是**觉醒前基础形态**,`c3=1,c4≥1` 才是觉醒后替换版;
  Addition 只有 `c4≥1`。判断"觉醒行" = `(c3=1 且 c4≥1) 或 c3=2`。leader 表 c1/c2 同理。)
- `c5` trigger 0=Instant瞬发 1=During持续 2=Opening开幕

**leader_ability**(共 124 列):

- `c0` string_id 文本键
- `c1` awake_kind 0/1/2(同左)——**没有 unisonable 列!**
- `c2` awake_level
- `c3` trigger 0/1/2

**ability_soul**(共 123 列):

- `c0` slot 槽位
- `c1` learn_level 习得等级
- `c2` trigger 0/1/2

**equipment_enhancement_ability**(共 126 列):

- `c0` slot
- `c1` learn_level
- `c2` max_power_level
- `c3` power.power1 强化值SLv1
- `c4` power.first_max 强化值满级
- `c5` trigger 0/1/2

**ex_ability**(共 125 列):

- `c0` string_id
- `c1` unisonable true/false
- `c2` 槽位/档位 1-5
- `c3` battle_power 战力值

## §4 单位与取值规则

| 规则 | 说明 |
|---|---|
| 千分比 | 强度类 `1000 = 1%`(60000 = +60%) |
| 计数阈值 | threshold/number `100000 = 1 次`(600000 = 6 次/6 人) |
| 帧(frame/cooltime 对) | `100000 = 1 帧`,60帧=1秒(18000000 = 180帧 = 3秒) |
| instant_delay | 原始帧数(不放大) |
| SLv 两端值 | 每个数值字段一对列:power1=技能等级1、first_max=满级,线性插值(`AbilityPowerValue.resolve`) |
| 空值 | 空串=未使用;`(None)`=显式空;布尔列 true/false |
| HP% 阈值 | HpHigh/HpLow 等的 threshold 千分比:1000=1% |

## §5 常用修改配方(列号均为 ability 表;leader 表自行 -2)

### 5.1 主位限制(解除/添加)
- `c1 unisonable`:`false`=仅主位生效(Ⓜ图标),`true`=协力位也生效。
- 前置条件枚举 `202 OwnerIsMain`(自身在主位)/`203 OwnerIsUnison`(自身在协力位)/`201 UnisonWithMain`(与指定角色组主位协力,组在+5列)。
- **解除主位限制** = c1→true,且把 c6/c13/c20 里的 202 → 0(Always)。工具:GUI mainpos 页或 `wf_patch_ability_main_position.py`。
- **加主位限制** = 反向操作;只想限制协力位就用 203。

### 5.2 属性共鸣触发(添加/去除/移植)
- 编码 = 前置条件 `2 Member`(队伍成员计数):kind=2 · threshold=`600000,600000`(6人=全队含协力) · character_groups=元素组。
- ability 表:`c6=2, c9=600000, c10=600000, c11=Red`;leader 表:`c4=2, c7=600000, c8=600000, c9=Red`。
- 元素组取值:`Red`火 `Blue`水 `Yellow`雷 `Green`风 `White`光 `Black`暗(CharacterGroup.Element)。
- 5共鸣改 `500000`;也可用种族组(如 `Dragon`)做“种族共鸣”。
- **多属性编成条件** = `5 ElementVariety`(threshold=属性种类数×100000)。
- **去共鸣** = kind→0(Always) 且清空该块 threshold/角色组列。GUI `transplant_line` 的 `element_resonance` 预设已按目标表类型自动换算(ability 清 c9/c10/c11+c6→0;leader 清 c7/c8/c9+c4→0),`precondition2/3` 预设同理整块清除并把 kind 回 0。

### 5.3 给自身还是全体(目标列)
- 瞬发效果目标 = `c48`;持续效果目标 = `c110`(leader:46/108)。取值(§6.6 全表):
  `0`=自身 `1`=除自身全员 `2/3/4`=队长/2号/3号位 `5`=全队 `6`=协力全队 `7`=触发者 `8/13/14`=多球 `9-12`=HP最低者系列。
- 目标+1列(c49/c111)=目标·角色组过滤:如 target=5(全队)+`Red` = “火属性全体”(莉莉丝队长技 line2 实例)。
- 注意:词条强度对全队生效时数值不变,纯改目标即可;瞬发 Condition 类效果目标改 5 即全体上 buff。

### 5.5 角色独有特殊效果的实现机制
- **固有状态(unique_condition)**:角色专属计数状态。赋予=`461 ConditionUnique`,消耗=`525 ConsumeUniqueCondition`,
  前置判定=precondition `188 ConditionCountUnique`(阈值×100000=层数),瞬发前置消耗=precontent 2/1。
  `unique_condition_id` 是全局编号(如莉莉丝=15),移植时**所有引用行的 uid 列要一起搬**,否则叠层/消耗互相找不到。
- **追加技能动作** `629 InvokeSkill`:c70=技能键(string_id)、c71=`battle/action/skill/action/ability_skill/...` 动作路径——
  可让任意角色释放另一个角色的技能动画+判定(莉莉丝队长技 line5 实例)。
- **强化弹射覆盖** `722/419 PowerFlipOverride`:c82-84(pfo.id/levels/description_id)把强化弹射整个换成专属形态。
- **切换技能形态** `536/704-708 ChangeSkillFlag1-6`:按 string_id 切换角色技能(变身/多形态角色)。
- **多球** :multiball_group_id 列 + 目标 `8 Multiball`/`13 ByGroupId`/`14 ByCharacterGroup`。
- **倍增触发**(c75-c81):效果强度随强化弹射/技能发动/Fever 递增(c74 初倍率,c76 每次追加)。
