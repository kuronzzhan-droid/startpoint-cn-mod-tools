# WF 单机版 · 角色数据逆向与修改指南

> 面向 startpoint-cn（世界弹射物语 CN 服务端模拟器）二次开发。
> 目标:把角色数据"逆向到可修改"的状态,并给出改词条 / 数值的落地路径。
> 本指南所有结论均在你的工程目录 + 上传的 `WFdebug单机版.apk` 上实测确认。

---

## 一、总体架构:角色数据分两层

游戏客户端是 **Adobe AIR / AS3**(`assets/worldflipper_android_release.swf`),战斗与数值计算在客户端本地完成;服务端(startpoint-cn)只负责账号、拥有关系、养成进度等。因此"角色数据"分布在两层:

| 层 | 位置 | 谁用它 | 是否已解码 | 改它影响 |
|---|---|---|---|---|
| **① 服务端逻辑层** | `assets/*.json`、`assets/cdndata/*.json` | Node 服务端逻辑 | ✅ 已是 JSON | 抽卡池、编队、养成结算、身份/文本词条 |
| **② 客户端主数据层** | 单机手机数据包内的内容寻址资源库(`.../production/android_upload/<xx>/<hash>`) | AIR 客户端本地读取 | ⚠️ 需解开(见第四节) | 玩家实际看到的**基础数值(HP/ATK)**、技能实数值、战斗表现 |

关键结论:**角色的身份和全部文本词条**(名字、称号、技能名、技能描述、声优)已经在 ① 层解码好,可直接用本工具改;**基础数值(HP/ATK 成长)** 属于 ② 层,需要用定位器在数据包里找到对应主数据表再改。

## 三、编辑器工具:`wf_char_editor.py`

对 ① 层身份 + 文本词条做安全的导出 / 编辑 / 写回,未暴露字段按下标原样保留。

```bash
# 1) 导出(生成好读的 JSON + 便于 Excel 批量改的 CSV)
python wf_char_editor.py export --repo <startpoint-cn目录> --out ./edit

# 2) 编辑
#    - 改长文本/技能描述  -> edit/characters_editable.json
#    - 批量改身份字段     -> edit/characters_overview.csv(Excel 可开, 元素用中文名)

# 3) 写回(自动把原文件备份成 *.bak)
python wf_char_editor.py apply --repo <目录> --edited ./edit/characters_editable.json
#    或用 CSV:      --edited ./edit/characters_overview.csv
```

可编辑字段:`name / name_en / description / title / skill_name / skill_desc /
skill_plus_name / skill_plus_desc / leader_title / cv / rarity / element / race /
gender / role`。

**实测**:改玛丽娜(111002)名字 + 技能描述 + 稀有度,写回后字段数保持 12/37 不变,`.bak` 可随时还原。

改完 ① 层后,让改动生效:重启服务端(`bash scripts/start-cn.sh`),客户端下次 `/load` 读取即更新其经由服务端下发的部分。

## 四、② 层:客户端主数据(基础数值 HP/ATK 所在)

> **✅ 2026-07-05 已破解并落地 GUI**:HP/ATK 在 `master/character/character_status.orderedmap`,
> 是**嵌套 orderedmap**——外层键=角色ID(行**原样存储**内层二进制,不再 zlib),
> 内层键=等级断点(`1/10/80/100`),行=zlib CSV `"hp,atk"`(**列0=HP,列1=ATK**,
> 对照反编译 `CharacterStatusValues.as`:`hp=parseInt(row[0]); atk=parseInt(row[1])`)。
> 客户端对断点排序后二分查找,等级间**线性插值向上取整**(`CharacterBaseStatusLogic.getParameterAt`)。
> 505/505 全量圆环验证通过;实测 168 个角色内层键序为 `10,1,80,100`,写回保持原键序。
> 读写实现:`wf_mod_tool.py` 的 `load_status_table / decode_status_row / encode_status_row /
> write_status_table`;GUI「基础数值」tab 可直接编辑 + 倍率缩放(改后 adb 同步)。
> **✅ 觉醒加成表 `character_awake_status.orderedmap` 也已破解**(36 键平表,zlib CSV 单行 2 列):
> 键=角色ID(36 个有觉醒魔晶板的角色),行=`"atk_plus_value,hp_plus_value"`——
> **列序与 character_status(hp,atk)相反!**(对照 `CharacterAwakeStatusValues.as`:
> `atk_plus_value=row[0]; hp_plus_value=row[1]`)。
> 面板公式(`BattleCharacterLogic`):`加成 = 已点亮觉醒大节点数 × plus_value`,
> 即表值是"每点亮 1 个觉醒大节点"的增量(如瓦格纳 `31,0` = 每大节点 ATK+31)。
> GUI「基础数值」tab 下方可直接编辑(仅限已有 36 键,不允许新增)。

### 4.1 最重要的逆向发现:**不是加密,是 raw-DEFLATE**

数据包里 `.../production/android_upload/<xx>/<hash>` 那些"看起来加密"(统一 `ec 9d…` 开头)的文件,**并非加密**,而是 **raw DEFLATE 压缩流**:

```python
import zlib
raw = zlib.decompress(open(path, "rb").read(), -15)   # -15 = raw deflate, 无需密钥
```

解开后是真实资源:ATF 贴图 / PNG / Ogg 音频 / 字体 XML / **orderedmap 主数据**。
主数据是 **制表符分隔的 orderedmap 文本**(实测在 bundle 里见到 `timelines / sequences / beginend` 这类表头 + Tab 分列)。

资源库 → 文件名映射:清单 `EntityLists/10939-android_medium.csv`,每行
`zipPath, version, size, contentHash, tag`;`unzip_cdn.ts` 把 zip 解开后按 `contentHash` 落到内容寻址目录。

### 4.2 定位器工具:`wf_scan_masterdata.py`

```bash
# 统计某资源库类型分布 + 列出"疑似主数据"候选
python wf_scan_masterdata.py scan --store <单机数据包>/WorldFlipper/dummy/download/production/android_upload

# 按角色ID/代号搜索,精确定位角色数值表
python wf_scan_masterdata.py find --store <资源库> --needle 111002 --needle pirates_girl

# 解开单个文件查看/导出
python wf_scan_masterdata.py dump --file <hash文件> --out out.bin
```

> 注:上传包里已解压的子集(`android_upload` 抽样 996 个、APK 内 `bundle.zip`)分别是**贴图**与**UI/字体/战斗时间轴**;角色**数值主表**在完整的 10GB `单机版数据包.zip` / `assets.rar` 里,用上面的 `scan` / `find` 即可捞出。

### 4.3 改 ② 层的完整回环(下一阶段落地)

1. `find` 定位角色数值主数据文件 → `dump` 解成文本;
2. 编辑 orderedmap(Tab 分隔)里的数值列;
3. 重新 `zlib.compress(data)[2:-4]` 或 `compressobj(wbits=-15)` 压回 raw-deflate;
4. 计算新内容 hash → 更新 `EntityLists/10939-android_medium.csv` 对应行的 `contentHash` 与 `size`,把文件放到内容寻址目录;
5. 客户端 `isFullPackage` / `info.json.version` 校验通过后加载新数值。

> 该回环所需的 orderedmap 精确列定义,建议先用 `find` 抓到一张角色数值表样本,再据实际列头补一份"数值字段字典"(和第二节身份表字段字典同法)。这是做"新模式/自定义数值"的基础。
