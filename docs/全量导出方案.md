## 方向一:从代码抓全部路径(命名)

原理:每个文件名 = `SHA1(逻辑路径 + 盐)`,盐已知(`K6R9T9Hz...`)。逻辑路径来自客户端代码:
一部分是**字符串常量**,一部分是**模板 + ID 拼接**(如 `character/{id}/square_0`)。

步骤(本地):

1. 抽常量池 + 类名(秒级,已做成工具):
   ```
   java -jar ffdec_26.2.1\ffdec.jar -dumpAS3 wf.swf > classlist.txt
   python mod-tools\wf_extract_paths.py 弹国服\wf_M358262.apk --classlist classlist.txt
   ```
2. 枚举 ID 撞模板:从主表拿全部 character_id / equipment_id / quest_id / enemy_id,
   套代码里的路径模板(`character/{id}/...`、`equipment/{id}/...`、`quest/thumbnail/...`),
   逐一 `SHA1(path+salt)` 撞库。你机器内存大,可一次性把 11 万 hash 全撞。
   —— 模板清单从方向二的反编译代码里 grep `"character/"`, `"equipment/"`, `"quest/"` 等前缀得到。

产出:尽可能完整的 `逻辑路径 → 存储位置` 全表。

## 方向二:按代码逻辑写解码器,全量解密导出

数据表编解码**已经逆向完成**,在 `mod-tools/wf_mod_tool.py`:

- 文件结构:`[u32 索引压缩长度][zlib(索引)][各行 zlib 数据]`;索引里 `row_offset` 是**行尾**偏移。
- 每张表配一个 **AMF3 schema**(列名/类型/枚举),schema 文件也在 store 里(见 `load_ability_schema`)。
- 已实现:`read_orderedmap_file` / `build_orderedmap` / AMF3 schema 解析 / CSV 导出。

**全量导出脚本**(本地跑,把每张表解成 CSV/JSON):

```python
# export_all_tables.py —— 遍历 DataCatalog.csv,每张表解码导出
import csv, sys
sys.path.insert(0, 'mod-tools')
import wf_mod_tool as core
from pathlib import Path
store = Path(r'弹国服\WorldFlipper\dummy\download\production\upload')
out = Path('export'); out.mkdir(exist_ok=True)
for r in csv.DictReader(open('mod-tools/DataCatalog.csv', encoding='utf-8-sig')):
    loc = r['存储位置']; name = (r['逻辑路径(已知)'] or loc).split('  ')[0]
    safe = name.replace('/', '__').replace('.orderedmap', '')
    try:
        om = core.read_orderedmap_file(store / loc, name)
        with open(out / (safe + '.csv'), 'w', newline='', encoding='utf-8-sig') as fh:
            w = csv.writer(fh)
            for k, t in om.text_rows().items():
                for line in core.read_csv_lines(t):
                    w.writerow([k] + line)
    except Exception as e:
        print('skip', loc, e)
```

配 schema 还能把列名也带上(参照 `wf_gui.py` 的 `export_annotated`,已对 ability 表实现)。

二进制资源(图片/音频)按类型还原:多为标准 PNG/加密 PNG/音频,需从 `pinball.asset.**` 反编译看读取逻辑(是否有 XOR/头部混淆),再批量转。这一步依赖方向二的反编译结果。

## 反编译(方向一/二都要,本地做)

```bat
:: 全量 AS3 伪代码,大堆,后台。本地无 45s 限制,几十分钟到数小时。
java -Xmx16g -jar ffdec_26.2.1\ffdec.jar ^
  -export script D:\wf_as3_out 弹国服\...\wf.swf
:: 只要逻辑参考包,更快:
java -Xmx16g -jar ffdec_26.2.1\ffdec.jar -selectclass "pinball.**" ^
  -export script D:\wf_as3_out wf.swf
```

重点包:`pinball.master.generated`(表结构+路径,补那 213 张未命名表)、
`pinball.asset`(资源读取/解密逻辑)、`pinball.abilityDescription`(词条描述拼装 → 面板级中文备注)。

> 沙盒里实测:每个混淆类完整 AS3 约 15–20s 且默认堆 OOM;本地 `-Xmx16g` 后台跑即可。
> 把反编译结果 clone/放进工作文件夹,我再帮你从代码里批量 grep 路径模板、补全命名与解码逻辑。
