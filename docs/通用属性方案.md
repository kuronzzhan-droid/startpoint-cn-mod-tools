# 通用属性(OmniElement)方案

逆向结论 + 落地状态的固化档案(2026-07-12,含当日实测修正)。
目标:让指定角色**计入任意元素的共鸣/编成/[限X属性]条件**。

## ⚠️ 0. 重大修正(2026-07-12 实测):element=6「真无属性」不可行

> 阿尔克 element 改成 6 后,游戏内**查看/进战即崩**(C7050),已回滚。
> **Colorless(6)是敌人/boss 专属元素,不能给可玩角色用**:
> - `forceUncolorless(6)` 硬抛(ClientError 10014),ability 描述/内容构造会走到;
> - 战斗连锁 `SkillChainManagerImpl` 用 `ElementKindValue.LENGTH=6` 建数组按元素索引 →
>   element=6 **越界**(合法 0–5);
> - `element-colorless` 补丁只修 UI 层,**救不了这两处逻辑层崩溃**。
>
> **修改器已彻底禁止给可玩角色写 element=6**(`save_char_fields` 硬拦截,资料页元素下拉
> 去掉「通用」项)。~~形态 B(真·通用属性)~~ 作废。

## 1. 唯一可用形态:A. 通用共鸣(OmniElement 标签)

| 数据侧 | 客户端补丁 | 效果 |
|---|---|---|
| character 表 c5 挂 `OmniElement` 标签(**element 保持原值不动**) | `client-patch/omni-element`(3 处,含副位) | 角色**保留真实元素**(伤害/克制/属性球/UI 全部正常),但计入**任意元素**的共鸣计数/编成数/[限X属性]效果目标/触发来源 |

**这满足「能满足任意共鸣」,但不是「无属性」**——角色仍有真实元素打伤害、吃克制。
引擎不支持可玩角色无属性,这是能做到的上限。修改器「一键通用共鸣」/「共鸣通用」开关
= 只加 c5 标签(安全),无补丁时标签零效果(无害)。

## 2. 数据层逆向结论

### 枚举 6 = Colorless(**敌人专属**,不要给可玩角色写)

- `ElementKind_Impl_.toColorlessable()` 恒等,**只**是「不炸的取值口」;但很多路径不走它,
  而走 `forceUncolorless`(硬抛)/连锁数组下标(越界)——见 §0/客户端补丁矩阵。
- `EnemyElementKind.Colorless` 官方存在——**这是敌人/boss 的无属性**,和可玩角色不同类型。
- 词条角色组 token 认 `Colorless`(条件层),但可玩角色 element=6 在**逻辑构造层**就崩了。
- 教训:表能存 6、名义枚举有 6,**不代表可玩角色能用 6**。已实测崩溃并禁止。

### character_tag(c5)——OmniElement 走这里(安全)

客户端**不做表校验**,塞未知 token 无害 → `OmniElement` 自定义标签做逐角色开关,
补丁只认这个标签;没打补丁时标签零效果(可先埋数据再换客户端)。

### 元素组匹配的三处独立实现(omni-element 补丁点)

1. `pinball/common/data/character/BattleCharacterLogic.as` `matchCharacterGroup`(战斗内)
2. `pinball/common/data/battle/squadMember/SquadMemberSource.as` `matchCharacterGroup`(编队/共鸣 UI 主位)
3. `SquadMemberSource.matchUnisonCharacterGroup`(**副位**独立实现,Element 分支走
   `matchElements(unisonElements,…)`,`unisonCharacterTags` 官方现成)

各加一行 `|| (unison)characterTags.indexOf("OmniElement") != -1`,见 `client-patch/omni-element.md`。

## 4. 修改器落地(wf_gui,现状)

- **一键通用共鸣 / 共鸣通用开关**:「角色资料」页,`POST /omni_convert` 或
  `/omni_element/set` = **只挂 OmniElement 标签(c5),不改 element**。
  dry-run 预览 → 确认 → 自动备份 + ②层进待发布。已开启时 no-op。
  dry-run 附**属性配对检查报告**:扫词条/队长技行级描述里的六色引用,逐条列出——
  **均保留原样**(角色自身伤害/克制按原元素不变,共鸣由补丁兜住)。
- **硬拦截**:`save_char_fields` 拒绝 element=6/通用;资料页元素下拉去掉「通用」项。
  (`.el.通用` 彩虹角标 CSS 保留,仅用于万一有 boss 复用体的显示。)

## 5. 生效链路

1. 修改器一键 → c5 标签写入,②层进 pending;
2. 「发布并重启游戏」(= `wf_publish` + 重启)→ 数据侧生效;
3. 客户端补丁(一次性,只需 omni-element):FFDec 导出 AS3 →
   `apply-omni-element.sh`(3 处)→ 回封 SWF/重签 APK(见 `client-patch/README.md`)。
   无补丁时标签零效果(无害),可先埋数据。

## 6. 完整判定矩阵(2026-07-12 全量逆向 + 实测修正)

| 位点 | 判的是谁的元素 | 用 Form A(element 不变) |
|---|---|---|
| **`forceUncolorless(6)`** | 自身/ability 元素 | 💥 element=6 才会触发,Form A 不改元素→**不触发** |
| **`SkillChainManagerImpl` LENGTH=6 数组** | 自身 | 💥 element=6 才越界,Form A→**不触发** |
| matchCharacterGroup ×2 + Unison | 自身 vs 组条件 | ✅ OmniElement 标签计入任意色(主位+副位),omni-element 3 处补丁 |
| ConditionSlot 抗性/减伤、对X敌加伤、限X多球 | 攻击方/敌方/球体元素 | 与自身元素无关,照常 |
| 自身伤害/克制/属性球/UI | 自身元素 | 按**原元素**正常(Form A 保留元素) |

**推论**:Form A(标签,不改 element)对**任意角色都安全**——绕开了 element=6 的两处
致命点,只借 c5 标签让角色计入任意元素共鸣。代价:角色仍有真实元素(非「无属性」)。

- **想要真无属性伤害**:引擎不支持可玩角色 element=6(会崩),无解,别追求。
