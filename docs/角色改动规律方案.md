## 1. 五张词条表列布局(移植前先认表)

**同构 body,只差表头长度**(全表 §2)。绝对列号 = 块基址 + 块内偏移(§3)。

| 表 | 列数 | 表头(c0..) | 块基址偏移 |
|---|---|---|---|
| **ability**(角色词条) | 126 | string_id, unisonable, 类别串, awake_kind, awake_level, trigger | 基准 |
| **leader_ability**(队长技) | 124 | string_id, awake_kind, awake_level, trigger(**无 unisonable!**) | ability **−2** |
| **ability_soul**(能力魂) | 123 | slot, learn_level, trigger | −3 |
| **weapon_ability**(武器,=equipment_enhancement_ability) | 126 | slot, learn_level, max_power_level, power.p1, power.max, trigger | **与 ability 同列号** |
| **ex_ability** | 125 | string_id, unisonable, 档位, battle_power, trigger | −1 |

关键块基址:precondition1 = ability c6 / leader c4;instant_content = ability c47 / leader c45;
during_content = ability c109 / leader c107。**跨表移植必须按此重排列,不能整行硬拷。**

## 2. 五类改动的标准做法

### 2.1 技能能量(action_skill)
嵌套表,外层键=code_name,内层键 `1`基础/`2`＋进化技。面板"技能能量" = 内层 **c5**(满级值),c4=SLv1。
改法:两列都设目标值 = 拉平。工具:`GET/POST /api/mod/skill_energy`。

### 2.2 队长技词条:修改 / 移植
- **改数值**:直接改该行数值列(强度=千分比 1000=1%,帧×100000=1帧,次数×100000=1次)。
- **整段换**(换成别人的队长技):`copy_leader_to_leader`(leader→leader 同表整行替换,可连描述键)。
- **追加单个效果**(保留原队长技 + 加一条):`transplant_line`(源行→目标 append)。**见 §3 铁律**。

### 2.3 角色词条:增 / 删 / 改
- **增**:`append_ability`(整词条所有行追加)/ `transplant_line`(单行追加)。
- **改**:`/rows/save`(逐字段改)。
- **删**:`delete_line`(删某行)。
- 一个词条键可多行(合法到 6+ 行),每行是一个独立效果或 SLv 端点。

### 2.4 主位限制开关
词条 `c1 unisonable=false`(仅主位Ⓜ)或前置枚举 `202 OwnerIsMain` = 主位限制。
**解除** = c1→true 且清 202→0;**添加** = 反向。工具:`/mainpos`(全局)或 `/mainpos_one`(单词条开关)。

### 2.5 武器词条(weapon_ability)
和角色词条同 schema(126 列),表头不同(c0=slot,c5=trigger)。读写复用 ability 逻辑,发布用 `weapon_ability`。

## 3. 移植铁律(踩了 6 次 U0000 换来的)

1. **同属性!同属性!同属性!** 源角色和目标角色元素必须一致。跨属性移植(如风效果塞进火角色)→ 客户端生成描述时元素图标/文案解析不出坏串 → `checkDescription` 抛错 → **U0000 崩**。风罗尔夫加风效果一次过;火罗尔夫加风效果崩了 4 次。
2. **共鸣型效果别去共鸣**:去掉前置(kind→0)后描述生成会得到空/null/NaN → 崩。要落地到别的角色,**保留共鸣前置,只改元素列**(leader c9 / ability c11:Green↔Red…)跟目标角色对齐。
3. **统一 string_id**:追加的行 sid 改成和目标词条其它行一致(单一 sid),避免混合 sid 描述错乱。
4. **跨表移植要重排列**:leader(124)↔ability(126)列号差 2、表头不同。leader 行原样塞进 ability 键 = 整体错位 2 列 → 枚举读到垃圾 → 崩。要么同表移植,要么按 §1 列图重排。
5. **c1 分表对待**:ability 的 c1=unisonable(可设 true),leader 的 c1=awake_kind(设 "true" 会毁行)。工具已按目标表自动处理。
6. **找已有该效果的角色行当模板**:别凭空构造。要"队伍充能"就抄一条现成的队伍充能词条(如操里能力5),改数值+元素+sid;要"冲刺间隔缩短"就抄芙拉菲。

## 4. 效果做不到的边界(别硬试)

- **技能本身加效果**(如给技能加贯穿/最大速度固定):效果在编译好的 ActionDsl 程序(`program_path`)里,数据改不了。替代:做成"自身发动技能时赋予…"的队长技/能力,或整换成带该效果的角色技能(换 program_path)。
- **直接攻击计算次数**:游戏只有 DirectAttack**1/2/3**(计1/2/3次),**没有 4/5 次**。
- **不分属性全体充能**:队伍充能词条一律按属性过滤,没有"全属性"组;风队做成"队伍风角色"即等效全体。

## 5. 数值单位速记

千分比 `1000=1%`(强度/阈值)· 帧 `×100000=1帧`,`60帧=1秒`(30秒=180000000)· 次数 `×100000=1次`
(20combo 阈值=2000000)· threshold 计数 `100000=1次`。SLv 两列 power1(SLv1)/first_max(满级),游戏线性插值。

| 词条逐字段改 | `POST /rows/save` |
