# 角色资产(立绘/图标/语音)与全角色替换 · 逆向结论与可行性(2026-07-06)

## 1. 资产体系逆向结论(已验证)

| 项 | 结论 |
|---|---|
| 存储根 | `upload`(通用)/`medium_upload`(大图:立绘、cut-in)/`android_upload`(平台),三根并列于 production/ 下 |
| 文件定位 | 文件名 = `sha1(逻辑路径 + 盐)`,三根同规则(盐即 wf_mod_tool.SALT) |
| PNG 混淆 | 仅魔数 8 字节:存储态 `89 70 6E 67`("png"小写) ↔ 标准 `89 50 4E 47`。fileFaker/PNGConverter 原样移植到 `wf_assets.py`,往返字节一致 |
| MP3 混淆 | 逐帧把帧头首字节 `0xFF↔0x7F`(sync 2047↔1023),ID3v2/v1 跳过;**仅 CBR/Layer3**。已移植,实测 158 帧往返字节一致 |
| 发布通道 | 服务端扫 `archive-{common,medium,android}-diff` 三目录;medium 包内路径 `production/medium_upload/xx/hash`(官方包实测)。`wf_publish.py` 已支持 pending 前缀 `medium:`/`android:` 自动分包 |

## 3. 技能效果(ActionDsl)编辑结论

- 文件:`<program_path含$>.action.dsl.amf3.deflate` = raw-deflate + AMF3 命令树
  (ActionDsl→Block→Command→StopBall/CreateHitArea/Rectangle…)。
- `wf_dsl.py`:AMF3 子集解析器(带引用表)提取全部 int/double 叶子(偏移+语境);
  **原地补丁**:U29 整数用非规范前导 0x80 补位到等长,double 直接覆写 8 字节 →
  不需要完整序列化器,结构零风险。数值超出原字段字节数会拒绝。
- GUI「技能·倍率」每段技能有「效果参数」面板;**增删效果命令 = 不支持**(需重排序列化),
  替代方案:「整技能替换」换模板后再调数值。

## 4. 整角色替换可行性 → ✅ 已可落地,两条路线

**路线 A · code_name 重定向(一字段全换)**:角色资料页把 `code_name` 改成另一角色的
code(①层 character.json + ②层 character 表同步),客户端所有资产(立绘/像素/语音/技能
动画)瞬间指向目标角色。数据(词条/数值/技能表键)不变。最快的"换皮"。风险:双角色同码
共存正常(本就是共享机制);改回即还原。

**路线 B · 逐项替换(完全自定义)**:
- 数据:词条/队长技/技能/数值/觉醒 —— 前几轮工具全覆盖 ✅
- 文本:名字/称号/描述/技能文本 —— 角色资料页 ✅
- 美术:立绘/cut-in/图标/像素图 —— 角色资产页 ✅(sprite sheet 需按 atlas 布局作图)
- 语音:逐条替换 ✅(清单覆盖度受采集限制;可按 words/<名>.mp3 规律补)
结论:**换皮/魔改现有角色已全流程可用**。

## 5. 全新角色可行性 → ⚠️ 理论可行,需两步工程

1. **数据层**(主要工作量):8 张表新增键(character / character_text / ability×6 /
   leader_ability / character_status / action_skill / 可选 awake 两表)+ ①层两个 json 追加。
   现有 GUI 写接口带"键必须存在"白名单(安全设计),需新增**克隆角色**功能
   (复制模板角色的全部表行到新 id/新 code_name,再改)。orderedmap 追加键在
   词条表已验证可行(append 行同理);嵌套表(status/action_skill)外层追加原样字节即可。
2. **获得途径**:客户端角色列表跟随服务端存档 —— 用 admin 后台/邮件把新 character_id
   发进存档。客户端对"未知 id"的容错(图鉴/剧情引用缺失)未实测,需金丝雀:先克隆一个
   id+资产全套,进游戏验证详情页/战斗/图鉴不崩,再铺开。
3. 资产:新 code_name 全套文件按 §2 模板落三根(sha1 命名),diff zip 能携带新文件
   (客户端直接落盘,机制已验证)。

**推荐实施顺序**:克隆角色工具 → 金丝雀验证 → 全新角色流水线。
