# WF 数据包 路径哈希 + 解密逻辑(从反编译源码逆向)

来源:`decompile/scripts/pinball/asset/**`(FileReader / AssetPathTools / MasterSource)。
`.filelist` 丢失时,靠这套规则从代码抓路径 + 内容解密,复原整个数据包。

## 1. 文件名哈希(AssetPathTools.getHashedPath)

```
规范化(p) = 把 [/\\]+ 合并成 /,再去掉开头的 /
文件名     = SHA1( 规范化(路径 + 扩展名) + "K6R9T9Hz22OpeIGEWB0ui6c6PYFQnJGy" )
存储位置   = 文件名[:2] / 文件名[2:]
```

**要点:路径和扩展名是拼在一起后整体哈希的**(如 `character/白虎/pixelart/sprite_sheet.png`)。

## 2. 去哪个 store 找(前缀不进哈希,只选文件夹)

`production/` 下有三个 store:`upload`(6.4G,基础)、`medium_upload`(3.2G,中等图)、`android_upload`(平台相关)。
FileReader.resolveFiles 按资源决定目录前缀,**前缀不参与哈希**,只决定去哪个文件夹:

- `.png` 且是可缩放资源:medium 画质 → `medium_upload`,small → `small_upload`;否则 `upload`。
- 平台相关资源(如 .atf 纹理):加 `android_` → `android_upload`。
- 所以同一个哈希文件名,可能落在 upload 或 medium_upload,撞库要三个 store 都查。

## 3. 扩展名(readXxx 方法)

| 资源 | 扩展名 |
|---|---|
| 主数据表 | `.orderedmap` |
| 图片纹理 | `.png`(或平台相关 `.atf.deflate`) |
| 音频 | `.mp3` |
| 帧动画 | `.frame` + `.amf3.deflate` |
| 影片 | `.movie` + `.amf3.deflate` |
| 时间轴/布局/图集 | `.timeline` / `.layout` / `.atlas` + `.amf3.deflate` |
| html/xml/css | 各自 + `.deflate` |

(`dataFileExtension = .amf3.deflate`,`compressedFileExtension = .deflate`;单机 intermediate 模式下是 `.json` 无压缩。)

## 4. 内容解密(fileFaker.converter,已在导出器实现)

- **PNG**:头 3 字节被 +0x20("PNG"→"png"),减回即原图。
- **MP3**:首字节最高位被清(0xff→0x7f),或明文 ID3。
- **AMF3**:zlib(可能带 4 字节长度前缀)解压 → AMF3 对象(动画/schema/config)。

→ `mod-tools/wf_export_assets.py` 已实现,**不需要路径就能按内容解密导出全部 11 万文件**。

## 5. 路径模板(从源码 grep 出的拼接式)

大部分资源路径 = `"前缀/" + 变量 + "/尾串"`,变量是**角色 code_name** 或**数字 ID**。例:

- 角色:`character/{code_name}/pixelart/sprite_sheet`、`/pixelart/special`、
  `/ui/full_shot_1440_1920`、`/ui/illustration_setting_sprite_sheet`、`/voice/{n}`、`/battle/floor` …
- 用 505 个 code_name × 这些尾串,已实测命中 2300 文件、覆盖 501/505 角色。
- 装备/关卡/敌人/道具同理(前缀 equipment/、quest/、enemy/、item/),尾串从各自 builder grep。

## 6. 抓取工具
