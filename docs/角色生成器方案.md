# 角色生成器与编辑器 · 设计方案(v2)

> v1 2026-07-15 设计;v2 2026-07-16 修订。目标:把「自制全新角色」从一次性人工流程固化为
> **可复用的生成+编辑工具**,全套美术/像素动画/语音资产经 AI 自动生成,数据层复用已验证链路。
> 实施者注意:本方案的每条"复用"都给出了源文件与函数名,不要重新发明已验证的轮子。
>
> **v2 变更概要**(相对 v1):
> ① **发布链路对齐 `wf_character_flow.py`**——生成器全阶段零直写 live store/.cdn/assets,
>    v1 的 S7"apply 直写"废除,改为组包进 flow workspace;初版走 `runtime_test`,
>    完整版走 production 发布(见 §7 S7/S8 与 `docs/角色包工作流.md`);
> ② **语音改 provider 抽象**:开源引擎(GPT-SoVITS 兼容 HTTP)为主,OpenAI TTS 为后备;
> ③ 新增**三种数据模式**:官方组装(assemble)/原创(create)/混合(hybrid)(§4);
> ④ 新增**两段式流水线**:初版(draft)→完整(complete)(§5);
> ⑤ 新增**调整循环**:整体调整+单项细节调整,自然语言指令定向重生成(§8);
> ⑥ 新增**一键生成 API**:`POST /chargen/oneclick` + CLI 同名子命令(§9)。

## 2. 目标与非目标

### 目标(v2)

用户在网页修改器新页签「角色生成」里,从一段中文概念描述(+可选参考图)出发:

1. 选模板角色 → 自动分配空闲 ID/code_name → 冲突预检;
2. LLM 生成全部身份文本(中英名/称号/描述/技能名与描述/队长技称号/台词稿 中文+日语);
3. gpt-image-1 生成基础/进化两张透明母版 → 确定性派生全部 UI PNG + cut-in ATF + 立绘定位/trim 同步;
4. 像素动画两种模式:**调色**(全自动,零风险)或 **AI 逐帧重绘**(保布局保时序,换外观);
5. 语音经 **provider 抽象**生成(开源引擎为主/OpenAI TTS 后备),ffmpeg 转 CBR + 严格校验;
6. 数据层三模式:**官方组装 / 原创 / 混合**(§4),LLM 起草+人工确认,哨兵校验强制;
7. **两段式**:初版(draft)最快路径上真机看效果 → 完整版(complete)逐项补齐(§5);
8. **调整循环**:任意阶段可下自然语言指令做整体风格调整或单项细节调整,只重生成受影响项(§8);
9. **一键生成**:单 API/单命令从概念直达初版组包+预检,发布永远人工确认(§9);
10. 每个资产可单独重新生成/手动上传覆盖/编辑提示词;逐序列 GIF 预览、语音试听;
11. 校验(37/37 完整度+尺寸+往返+QA 门)→ 组包 → `wf_character_flow` 预检/发布;
12. 全程可回滚(spec 修订链 + flow 反向增量回滚)。

### 非目标(v2,显式排除)

- **不做** flatomo 新骨架/新时间线/新 SFX(编译器未落地)。⚠ 与 v1 的差异:**特效"重皮"
  (reskin,克隆官方特效目录+AI 重绘贴图,布局零改动)在 v2 范围内**,骨架级"真·新特效"仍排除;
- **不做** 双形态/matched 资产/客户端基座 APK(赛瑞斯专属轨道);
- **不做** 数值平衡的 AI 自动设计——v2 允许 AI 起草词条/技能的**结构组装**(§4 模式 B),
  但数值默认取官方先例值,平衡微调深链现有页签(词条编辑/技能·倍率/效果构建器);
- **不生成** story/words/login 语音、剧情表情差分、剧情横幅(`wf_character_flow` 包校验
  明确排除这些路径,生成器也不得顺手复制);
- **不做** 卡池接入(种子页/卡池组配置)——发放走 admin 邮件,卡池留开放问题;
- **不改** 服务端 TS 代码、`web/pages/`、`src/routes/web/`、`web/public/`、admin SPA。

## 3. 总体架构

```
spec.json(唯一事实源,work/char_gen/<code>/)
   │
   ├─ wf_openai.py    OpenAI API 客户端(纯 stdlib,可注入假传输)
   ├─ wf_voice.py     语音 provider 抽象(local_http 开源引擎 / openai 后备,可注入假传输)
   ├─ wf_char_gen.py  生成引擎:阶段化流水线,纯函数+缓存,不 import wf_gui
   │     stages: plan → text → masters → ui → pixel → voice → data → qa → pack
   └─ wf_gui.py       编排:新端点 /chargen/*(含 oneclick/adjust/pack/flow)
         组包产物交给 wf_character_flow.py(status/preflight/publish 子进程,解析末行 JSON)
```

- **目录约定**(gitignore):
  - `work/char_gen/<code>/`:`spec.json` 规格 | `authoring/` 生成原始产物(母版/逐帧/原始
    TTS/**历史修订版**) | `build/` 编译产物(派生 UI/重组 sheet/CBR MP3) | `qa/` 预览与报告 |
    `cache/` API 响应缓存(按请求哈希);
  - `work/character_packs/<package_id>/`:`wf_character_flow` workspace(组包交付区,
    结构见 `docs/角色包工作流.md` §1),**生成器只写它的 `package/` 子树**。
- **spec.json** 记录:概念文本、参考图路径、模板 ID、新 ID/code、**mode(assemble/create/
  hybrid 槽位表)**、**tier(draft/complete)**、身份卡(发色/瞳色/服装/配色等不变量)、身份字段、
  逐资产的 prompt/seed/状态/哈希/**修订链**、台词表(路径+中文稿+日语+**声线卡**)、像素模式、
  **数据构建单(技能/词条/队长技/PF 逐槽位来源与内容)**、**adjustments[] 调整史**、QA 结果、
  组包/发布状态。编辑器每次改动即回写;CLI 可从 spec 无 UI 重建全部产物。
- **资产状态机**:`pending → generated → locked → packed`;调整循环可把任意项打回
  `dirty`(=pending 且带增量指令)。锁定(locked)是跨阶段依赖的闸门(如母版锁定才许跑 UI 派生)。
- **幂等与断点续跑**:每阶段先查 `cache/` 与产物哈希,已完成且输入未变则跳过;`--force` 重跑。
- **API Key**:`OPENAI_API_KEY` 环境变量(已配置,用户级),其次 `mod-tools/work/openai.json`
  (gitignore;字段 `api_key`/`base_url`/`models`/`tts`)。**必须支持 `OPENAI_BASE_URL` 覆盖**
  (代理/中转场景)。Key 缺失时页面显示配置指引,生成按钮置灰,其余(手动上传/编辑/组包)照常可用。

## 4. 数据模式(mode:assemble / create / hybrid)

数据层(技能/词条/队长技/PF)的构建策略,spec 顶层 `mode` + 逐槽位覆盖。

### 模式 A:assemble 官方组装(默认,最快最稳)

全部数据取官方现成件重新组合,**零新 DSL、零新资产、先例原则天然满足**:

- **技能**:整技能替换——从 action_skill 全库选(检索语料=`/skill_summary` 中文效果摘要+
  名称+元素),或保留模板技能;技能名/描述照常改(三层同步);
- **词条**:全库检索移植——检索语料=`/search_abilities` 行级中文描述,写入语义=
  `append_line_adapted`(自动适配元素/sid/觉醒门槛,U0000 铁律自动化);
- **队长技**:同词条(跨表 角色词条↔队长技 的 `as_key` 列重排已有);
- **PF**:c6 官方 5 种直选,或 PF 合成工坊组合已有种类(`/powerflip/compose` 语义);
- **战斗特效/演出**:随所选官方技能 DSL 自带,不缺资产。

**LLM 职责 = 构建建议单**:根据人设概念检索全库,对 技能/每条词条/队长技/PF 各给出
2-3 个候选(附中文理由与出处角色),人工在编辑器勾选确认后进构建单。数值沿用官方原值。

### 模式 B:create 原创

- **词条**:效果构建器全量枚举自由拼(触发 262/230 × 效果 724/422),LLM 起草 →
  **`_client_legality_problems` 哨兵硬校验**(空串哨兵/(None)/puller/门禁全套)→ 人工确认;
  InvokeSkill(629) 自动注册 `custom_ability_string`(C8601 预防);
- **技能**:命令级组装新 DSL——从全库 1024 技能的命令实例检索取件(`/skill_cmd_lib` 语义),
  `wf_dsl_sig.py` 签名表约束参数个数与枚举;产物为新逻辑路径的 DSL 文件(进组包,非直写);
- **特效重皮(reskin)**:克隆官方特效目录到新逻辑路径(贴图+atlas+`.parts` 骨架**布局字节原样**),
  AI 重绘贴图帧(管线同像素 repaint:裁 cell→放大→images.edit→缩回→量化→**IoU≥0.6 门**),
  新技能 DSL 内 `SpecifyEffectDirectly` 指向新路径。**硬边界:不做新骨架/新时间线**;
- **像素动画**:repaint(§7 S4);
- **风险控制**:原创产物属"无先例组合",**强制先走 draft(runtime_test)真机金丝雀**,
  真机通过才允许升 complete/production(§5)。

### 模式 C:hybrid 混合

spec 逐槽位声明来源,任意混搭:

```json
"mode": {
  "kind": "hybrid",
  "skill": "official",          // official | new
  "abilities": ["official", "official", "new", "official", "new", "official"],
  "leader": "official",
  "pf": "composed",             // official | composed | new_override
  "effects": "reskin",          // template | reskin
  "pixel": "recolor"            // recolor | repaint
}
```

典型用法:官方技能保稳 + 2 条原创词条点题 + 特效重皮换观感。校验按槽位来源分别走
A/B 的规则;含任何 `new` 槽位即视同模式 B 的风险等级(必须先 draft 真机)。

## 5. 两段式流水线(tier:draft → complete)

### draft 初版(目标:概念 → 真机可见,人工确认 ≤3 次)

**范围裁剪**:
- S1 全文本(便宜,一次到位);
- S2 仅基础母版(进化位暂用基础母版复用,spec 标记 `evolved_master: "reuse_base"`);
- S3 全量 UI 派生照跑(确定性裁切,不是瓶颈,且 37/37 需要);
- S4 recolor(全自动);
- S5 语音**跳过**(ally/battle/home 语音是建议项,不计入 37 门槛,见工作流 §1);
- data 阶段 = 模式 A 最小集:模板技能/词条原样,只换身份文本(或用户已确认的构建单子集);
- S6 QA(draft 范围)→ S7 组包 → `status` 37/37 → `preflight`。

**发布 = `runtime_test` 通道**:`delivery_mode=runtime_test`、`release_ready=false`、
口令 `DIRECT_REAL_TEST`。**每次 runtime_test 发布都需用户在界面显式授权**
(`user_authorized_direct_real_test` 由用户操作产生,生成器不得自动置 true)。
真机看效果 → 不满意进调整循环(§8)重生成 → 重组包重发。

draft 的意义:先让角色站在游戏里,再决定投入完整生成成本(repaint 40-80 帧调用、全量语音)。

### complete 完整版

在同一 spec 上逐项升级,每步可插调整循环:

1. 进化母版真实生成 + 进化侧 UI 重派生;
2. 像素 repaint(可选,默认保留 recolor);
3. 全量语音(S5,按模板 voice 清单逐条);
4. 数据层按所选模式完成构建单全部槽位;
5. S6 QA 全绿 → S7 重组包 → `rebase`(live 漂移时)→ `preflight` →
   **停服 production 发布**(口令 `PUBLISH_CHARACTER_PACKAGE`,保存 `snapshot_dir`)。

⚠ runtime_test 包**不能静默升级**为 production(工作流 §7)——complete 发布是全新走一遍
生产门禁;同名包升级用 `--installed-package-dir` 绑定(工作流 §5)。

## 6. 语音子系统(wf_voice.py,provider 抽象)

统一接口:`synth(text, lang, voice_card) -> wav bytes`,transport 可注入假实现。

| provider | 说明 | 定位 |
|---|---|---|
| `local_http`(默认) | 兼容 **GPT-SoVITS api_v2** 约定的本地 HTTP 引擎(`POST /tts`:text/text_lang/ref_audio_path/prompt_text/prompt_lang/…),`base_url` 可配(默认 `http://127.0.0.1:9880`);其他开源引擎经同一适配器接入 | 主力:中日双语、动漫声线、少样本声线克隆 |
| `openai` | `gpt-4o-mini-tts`,voice+instructions 取声线卡 | 后备:无本地部署时可用;日语动漫声线效果弱,QA 需人工试听把关 |

- **声线卡**(spec `voice_card`):provider、参考音频路径+参考文本(local_http)或
  voice+instructions(openai)、语速/情感默认参数、**`source_license` 来源授权声明**。
  同角色全部台词固定同一声线卡。
- **合规红线**:参考音频只允许 用户自录/明确授权素材/开源许可语料/AI 生成种子声线;
  **禁止克隆真人声优或未授权音源**;`source_license` 缺失 → voice 阶段拒跑。
- **输出管线不变**:provider 出 wav → ffmpeg `44.1kHz 单声道 96kbps CBR -write_xing 0` →
  `wf_assets.mp3_encode` 严格预检(不落 store,只验证能通过)。
- ffmpeg 探测:`WF_FFMPEG` env > PATH;引擎/ffmpeg 缺失时 voice 阶段整体禁用并给出指引
  (draft 不受影响——语音本就不在 37 门槛内)。
- 配置:`mod-tools/work/openai.json` 增 `"tts": {"provider": "local_http", "base_url":
  "http://127.0.0.1:9880", "defaults": {...}}`;GPT-SoVITS 部署步骤写入使用说明文档。

## 7. 各阶段规格

### S0 plan(模板与身份)

- 模板选择:任意现有角色(默认给出与目标概念同属性的 ★5 列表);
- ID 分配:扫 character 表键,推荐 `1{X}9999` 系列首个空闲(Kyle=119999、赛瑞斯=129999 惯例),
  可手工覆盖;16 表键冲突预检(复用 clone_character 的 dry-run **只读部分**);
- code_name:小写+下划线,校验未被任何 character 行使用、store 中 `character/<code>/` 无既有文件;
- **同时 `wf_character_flow.py init`** 建立组包 workspace(template-id/character-id/code-name/
  package-id 一次对齐),spec 记录 `package_id`;
- **模板资产清单动态探测**:`wf_assets.char_asset_manifest` 拿模板实际存在的资产与尺寸,
  **禁止硬编码尺寸表**(Kyle 的 illustration sheet 是 361×806,与文档 742×361 不同——以模板
  实测为准)。派生表 `DERIVATIVES` 从 `wf_kyle_canary.py` 提炼为共享常量(rect/mode 保留,
  size 运行时取模板实际尺寸)。

### S1 text(文本与台词)

- 一次 chat.completions 调用产出结构化 JSON(few-shot:从现有 character_text 抽 2 个真实角色):
  name/name_en/title/description/skill_name/skill_desc(+进化版)/leader_title/CV 显示名
  (标注"AI 声线",不写真人声优名);**身份卡**(发色/瞳色/服装/配色不变量)同批产出,供 S2/S4/调整循环;
- 台词表:**以模板 voice 清单为准逐条生成**(路径一一对应,不多不少;`character_speech` 已注册的
  home/ally 行克隆原样复制,资产路径对齐后表零改动);每条含中文稿+日语台词;
- 全部文本编辑器可改;文本落库走组包的表声明(三层同步语义等价 `save_char_fields`,
  但落盘目标是 workspace roots,不直写 live)。

### S2 masters(母版)

- gpt-image-1 生成 3 张透明背景母版:基础全身、进化全身、(可选)像素风格参考;
- prompt = 概念文本 + 身份卡拼装,进化版 prompt 显式要求同一人物;
- 参考图(用户提供的本地 PNG)走 images.edit 多图输入作风格锚;
- 每张可单独重生成;母版**锁定**后才允许跑 S3;
- **立绘是透明背景裁剪图**——不生成带场景背景的图;带背景素材仅可作 images.edit 的风格参考输入。

### S3 ui(UI 派生)

- 从锁定母版确定性派生模板清单里的全部 UI PNG:
  - `full_shot_1440_1920_{0,1}.png`:`fit_rgba` 到 1440×1920 → 按内容 bbox 计算
    `character_image`/`full_shot_image_attribute`/`trimmed_image` 三表行(**写进组包表声明**,
    算法复用 `/char_image_pos/save` 的实现,不要手写 trim 逻辑);
  - 其余小图:按共享 DERIVATIVES 表 `cover_rgba` 焦点裁切,尺寸=模板同名资产实测尺寸;
  - `illustration_setting_sprite_sheet.png`:按模板 atlas 逐 cell 填充,不改 atlas 本体;
  - `skill_cutin_{0,1}.png` → `wf_atf.py` 重编码 ATF(android 根,进组包 android root)。
- QA:输出一张 UI 马赛克总览图到 `qa/`。

### S4 pixel(像素动画)

两种模式,spec `mode.pixel`:

- **recolor(默认,零风险)**:LLM 从母版色板产出「模板色→新色」映射表(编辑器可视化可改),
  对模板 `sprite_sheet.png`/`special_sprite_sheet.png` 逐像素映射(精确色板+HSV 近邻两级,
  算法照抄 `wf_canary_skin.py`);atlas/frame/timeline **原样不动**。
- **repaint(AI 逐帧重绘)**:对模板 atlas 的每个 frame cell:
  1. 裁出 cell → 最近邻放大 8× → images.edit(输入:放大帧+像素风格参考母版;prompt:保持
     姿势与剪影,替换为新角色外观,限定色板,透明背景);
  2. 结果最近邻缩回原 cell 尺寸 → 量化到 ≤32 色 → alpha 阈值化;
  3. **帧级 QA 硬门**:新帧 alpha bbox 与原帧 IoU ≥ 0.6,越界/漂移帧标记"待重生成",不静默通过;
  4. 全帧就绪后按 atlas 原坐标重组 sheet(尺寸=模板 sheet 尺寸,布局零改动)。
  成本提示:两张 sheet 合计通常 40-80 帧,启动前显示帧数与预估调用数,要求确认。
- 两种模式 QA 相同:按 timeline 逐 tick 渲染每个序列的 GIF(帧号→图块规则:`{frame.name}%04d`
  缺号沿用前一张,tick 后缀=inclusive 绝对结束 tick——复现 `/pixelart_data` 预览器语义)到
  `qa/`,编辑器内逐序列播放。
- **特效重皮**(模式 B/C 的 `effects: reskin`)复用同一 repaint 管线,对象换成克隆到新路径的
  特效贴图目录;atlas/`.parts` 字节原样。

### S5 voice(语音)

见 §6。对台词表逐条:provider 合成 → 原始输出存 `authoring/` → ffmpeg 转 CBR 存 `build/` →
`mp3_encode` 预检;编辑器逐条试听/改词重生成/换语速语气重生成。draft 跳过本阶段。

### S5.5 data(数据构建,新阶段)

- 按 §4 模式产出**数据构建单**并物化为表行/DSL 文件:
  - assemble:候选检索(LLM)→ 人工勾选 → 生成适配后的表行(元素/sid/门槛适配规则同
    `append_line_adapted`);
  - create:LLM 起草 → 哨兵校验 → 人工确认 → 表行 + 新 DSL 文件 + `custom_ability_string` 新键;
  - 全部产物落 `build/data/`,S7 并入组包表声明;**不经 composer_apply 直写 live**。
- 目标属性≠模板 → 元素转换按 `/element_convert` 的规则集在**组包表声明内**完成
  (c3+token 翻译+content 元素列,规则复用其实现)。

### S6 qa(校验汇总)

- 资产清单 vs 模板清单差集为空;全部 PNG 可解码且尺寸=模板;MP3 全过严格校验;像素/特效 QA
  门全绿;数据行全过合法性哨兵;draft 允许 声音缺席+进化母版复用 两项豁免(报告标注);
- 产出 `qa/report.json` + 页面可读版;**报告不绿(含豁免规则),pack 按钮不亮**。

### S7 pack(组包,替代 v1 的"apply 直写")

1. 把 `build/` 全部产物按逻辑路径写入 flow workspace 的
   `package/roots/{common,medium,android,server}`;
2. 数据表改动写成 manifest 的表键/行声明(SHA-256/大小/语义占用,rebase 按声明行合并);
3. 生成/更新 `manifest.json`;**生成器唯一的 workspace 外写入=无**(live store/.cdn/assets 零接触);
4. 跑 `wf_character_flow.py status` → 37/37、三层声明一致、errors 空;
5. 缺项只补列出的逻辑路径,**不复制排除目录、不占位造假**(工作流 §8 故障处理)。

### S8 publish(发布与发放)

- **draft**:`preflight` → `runtime_test` 发布(口令 `DIRECT_REAL_TEST`,逐次用户显式授权);
- **complete**:`status → preflight → rebase(需要时) → 停服 → publish --confirm
  PUBLISH_CHARACTER_PACKAGE`,保存输出 `snapshot_dir`;
- 回滚 = `wf_character_flow.py rollback`(反向增量,见工作流 §6);spec 记录发布史;
- GUI 调 flow 一律子进程 + 解析**最后一行稳定 JSON**(`ok`/`release_ready`/`errors`/
  `next_command`/退出码),不解析提示文本;
- 发放:提示走服务端 admin 邮件(现有流程),页面给出角色 ID 与操作指引;不新写服务端代码。

## 8. 调整循环(整体调整 / 细节调整)

任意阶段之后可用,自然语言指令驱动,**只重生成受影响项**:

- `POST /chargen/adjust {spec_id, scope, target?, instruction}`
  - `scope=global`(**整体调整**):指令如「整体更暗黑」「喜剧化一点」「盔甲改成银白色」——
    LLM 把指令合并进身份卡/全局风格字段 → 按**依赖图**把受影响资产标 dirty → 逐项重生成;
  - `scope=asset|text_field|voice_line|data_slot`(**细节调整**):单目标,新 prompt=
    原 prompt+增量指令;如「觉醒立绘的表情更冷」「这句台词语速慢一点」「三号词条换成防御向」。
- **依赖图(dirty 传播规则)**:
  - 身份卡外观字段变 → 母版 dirty → 全部派生 UI+trim 定位+像素(recolor 映射表/repaint 参考)dirty;
  - 台词稿变 → 对应语音 dirty;技能名/描述变 → 文本表声明 dirty;
  - 数据构建单变 → 对应表行/DSL dirty → QA/组包 dirty;
  - 母版仅局部指令(如只改表情)→ 允许用户选择「只重生成受影响侧」(基础/进化)。
- **修订链**:旧版本产物移入 `authoring/history/<asset>/<rev>/`,spec 记 revision 链,
  编辑器可并排对比+一键回退上一版;`adjustments[]` 记录 时间/scope/指令/影响清单,全程可审计。
- 调整后重新走 S6 qa → S7 pack(哈希判定,未影响项不重做)。

## 9. 一键生成 API

- **HTTP**:`POST /chargen/oneclick`
  ```json
  {"concept": "中文概念描述", "ref_image": "可选路径", "mode": "assemble|create|hybrid槽位表",
   "tier": "draft|complete", "template_id": 111165, "element": "可选", "auto": false}
  ```
  返回 `spec_id`;后台顺跑 plan→text→masters→ui→pixel→(voice)→data→qa→pack(tier 范围),
  进度 `GET /chargen/status`。
  - `auto=false`(默认):在 ①母版锁定 ②数据构建单确认 ③发布授权 三个点暂停等人;
  - `auto=true`:①②自动通过(母版取首次生成,构建单取 LLM 首选),**仅在发布授权点必停**。
- **CLI**:`python mod-tools/wf_char_gen.py oneclick --concept "..." [--ref x.png]
  --mode assemble --tier draft [--template 111165] [--auto]`
- **发布永不自动**:oneclick 终点=组包+status+preflight 通过;`publish` 必须人工执行
  (CLI 手敲口令 / GUI 确认框输入口令)。
- 后台任务用独立线程+进度轮询,同时只跑一个(独立命名空间,不挤占 toolbox)。

## 13. 开放问题(实施中遇到按此决策)

- **flow 对生成器新增物的 manifest 语义占用**:新技能 DSL 路径/新特效目录/
  `custom_ability_string` 新键是否已被 `wf_character_flow` 的声明模型覆盖——实施首日先用
  最小 fixture 验证 `status`;不支持则先提 flow 需求,不绕门禁;
- **draft 迭代成本**:每次真机验证走一次 runtime_test 发布(停服要求以 flow 实际门禁为准);
  若迭代太重,方向是增强本地预览(GIF/立绘定位模拟),不是绕开 flow;
- OpenAI 图像透明背景:请求 `background: "transparent"` + `output_format: "png"`;被忽略时
  fallback 色键抠图(容差可调)并在 QA 标注;
- repaint 单帧成本过高 → 降级档「关键帧重绘+相邻帧色板迁移」(spec `pixel.repaint_keyframes_only`),
  v1 先做全帧;
- GPT-SoVITS 本地部署(显卡/权重/双语模型)写入使用说明;无引擎时 openai 后备的日语质量
  由人工试听验收,不设自动门;
- 模板语音清单若含 datamine 缺失文件 → 以 store 实际存在为准,差集报告 S0 就显示;
- 卡池接入(种子页/卡池组)v2 仍不做,发放走邮件;后续若做,应作为独立小工具而非生成器阶段。
