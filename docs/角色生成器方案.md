# 角色生成器与编辑器 · 设计方案(v1)

> 2026-07-15 设计。目标:把「自制全新角色」从一次性人工流程固化为**可复用的生成+编辑工具**,
> 全套美术/像素动画/语音资产经 OpenAI API 自动生成,数据层复用已验证的克隆链路。
> 实施者注意:本方案的每条"复用"都给出了源文件与函数名,不要重新发明已验证的轮子。

## 2. 目标与非目标

### 目标(v1)

用户在网页修改器新页签「角色生成」里,从一段中文概念描述(+可选参考图)出发:

1. 选模板角色 → 自动分配空闲 ID/code_name → 冲突预检;
2. LLM 生成全部身份文本(中英名/称号/描述/技能名与描述/队长技称号/台词稿 中文+日语);
3. gpt-image-1 生成基础/进化两张透明母版 → 确定性派生全部 UI PNG + cut-in ATF + 立绘定位/trim 同步;
4. 像素动画两种模式:**调色**(全自动,零风险)或 **AI 逐帧重绘**(保布局保时序,换外观);
5. OpenAI TTS 生成模板同清单的全部语音 MP3(经 ffmpeg 转 CBR + 严格校验);
6. 每个资产可单独重新生成/手动上传覆盖/编辑提示词;逐序列 GIF 预览、语音试听;
7. 校验(模板完整度+尺寸+往返)→ dry-run 预览 → 应用(克隆+替换+身份三层同步)→ 发布 → 提示邮件发放;
8. 全程可回滚(快照+char_delete+反向发布)。

### 非目标(v1,显式排除)

- **不做** Flatomo 新特效/新 SFX(编译器未落地,赛瑞斯计划里是硬门;技能特效随模板 DSL 原样克隆,不缺资产);
- **不做** 双形态/matched 资产/客户端基座 APK(赛瑞斯专属轨道,等其硬门通过后再并入);
- **不做** 词条/技能数值的 AI 自动设计(数值编辑深链到已有页签:词条编辑/技能·倍率/效果构建器;唯一例外:目标属性≠模板属性时调用已有 `/element_convert` 整体转属性);
- **不生成** story/words/login 语音、剧情表情差分、剧情横幅(包校验继续拒绝这些路径);
- **不改** 服务端 TS 代码、`web/pages/`、`src/routes/web/`、`web/public/`、admin SPA。

## 3. 总体架构

```
spec.json(唯一事实源,work/char_gen/<code>/)
   │
   ├─ wf_openai.py    OpenAI API 客户端(纯 stdlib,可注入假传输)
   ├─ wf_char_gen.py  生成引擎:阶段化流水线,纯函数+缓存,不 import wf_gui
   │     stages: plan → text → masters → ui → pixel → voice → qa
   └─ wf_gui.py       编排与落盘:新端点 /chargen/*,apply 阶段调用
         clone_character / replace_asset / save_char_fields / element_convert
```

- **目录约定**(沿用赛瑞斯计划的分层,均在 `work/char_gen/<code>/` 下,gitignore):
  `spec.json` 规格 | `authoring/` 生成原始产物(母版/逐帧/原始 TTS) | `build/` 编译产物(派生 UI/重组 sheet/CBR MP3) | `qa/` 预览与报告 | `cache/` API 响应缓存(按请求哈希)。
- **spec.json** 记录:概念文本、参考图路径、模板 ID、新 ID/code、身份字段、逐资产的 prompt/seed/状态/哈希、台词表(路径+中文稿+日语+语音参数)、像素模式、QA 结果、应用/发布状态。编辑器每次改动即回写;CLI 可从 spec 无 UI 重建全部产物。
- **幂等与断点续跑**:每阶段先查 `cache/` 与产物哈希,已完成且输入未变则跳过;`--force` 重跑。
- **API Key**:`OPENAI_API_KEY` 环境变量,其次 `mod-tools/work/openai.json`(gitignore;字段 `api_key`/`base_url`/`models`)。**必须支持 `OPENAI_BASE_URL` 覆盖**(代理/中转场景)。Key 缺失时页面显示配置指引,生成按钮置灰,其余(手动上传/编辑/应用)照常可用。

## 4. 各阶段规格

### S0 plan(模板与身份)

- 模板选择:任意现有角色(默认给出与目标概念同属性的 ★5 列表);
- ID 分配:扫 character 表键,推荐 `1{X}9999` 系列首个空闲(Kyle=119999、赛瑞斯=129999 惯例),可手工覆盖;16 表键冲突预检(直接复用 clone_character 的 dry-run);
- code_name:小写+下划线,校验未被任何 character 行使用、store 中 `character/<code>/` 无既有文件;
- **模板资产清单动态探测**:用 `wf_assets.char_asset_manifest` 拿模板实际存在的资产与尺寸,**禁止硬编码尺寸表**(Kyle 的 illustration sheet 是 361×806,与文档 742×361 不同——以模板实测为准)。派生表 `DERIVATIVES` 从 `wf_kyle_canary.py` 提炼为共享常量(rect/mode 保留,size 改为运行时取模板实际尺寸)。

### S1 text(文本与台词)

- 一次 chat.completions 调用产出结构化 JSON(给足 few-shot:从现有 character_text 抽 2 个真实角色做示例):
  name/name_en/title/description/skill_name/skill_desc(+进化版)/leader_title/CV 显示名(标注"AI 声线"即可,不模仿真人声优);
- 台词表:**以模板 voice 清单为准逐条生成**(路径一一对应,不多不少;含 `character_speech` 已注册的 home/ally 行——克隆会原样复制该表,路径相对于 `character/<code>/voice/`,资产路径对齐后表零改动);每条含中文稿+日语台词;
- 全部文本在编辑器可改;写入走 apply 阶段的 `save_char_fields`(已有三层同步:cdndata + character/character_text 表 + assets/character.json,技能名同步 action_skill)。

### S2 masters(母版)

- gpt-image-1 生成 3 张透明背景母版:基础全身、进化全身、(可选)像素风格参考;
- prompt 由概念文本+身份不变量(发色/瞳色/服装/配色,S1 时让 LLM 先产出"身份卡")拼装,进化版 prompt 显式要求同一人物;
- 参考图(用户提供的本地 PNG)走 images.edit 的多图输入作风格锚;
- 每张可单独重生成;母版审核通过(用户在编辑器点"锁定")后才允许跑 S3。

### S3 ui(UI 派生)

- 从锁定母版确定性派生模板清单里的全部 UI PNG:
  - `full_shot_1440_1920_{0,1}.png`:`fit_rgba` 到 1440×1920 → 按内容 bbox 写 `character_image`/`full_shot_image_attribute`/`trimmed_image`(全部走已有 `/char_image_pos/save` 与 `replace_asset` 的自动同步,自己不要手写 trim 逻辑);
  - 其余小图:按共享 DERIVATIVES 表 `cover_rgba` 焦点裁切,尺寸=模板同名资产实测尺寸;
  - `illustration_setting_sprite_sheet.png`:按模板 atlas(`/pixelart_data` 同族解码器读 `.atlas.amf3.deflate`)逐 cell 填充对应裁切,不改 atlas 本体;
  - `skill_cutin_{0,1}.png` → `wf_atf.py` 重编码 ATF(android 根)。
- QA:输出一张 UI 马赛克总览图到 `qa/`。

### S4 pixel(像素动画)

两种模式,spec 里 `pixel.mode`:

- **recolor(默认,零风险)**:LLM 从母版色板产出「模板色→新色」映射表(在编辑器可视化可改),对模板 `sprite_sheet.png`/`special_sprite_sheet.png` 逐像素映射(精确色板+HSV 近邻两级,算法照抄 `wf_canary_skin.py`);atlas/frame/timeline **原样不动**。
- **repaint(AI 逐帧重绘)**:对模板 atlas 的每个 frame cell:
  1. 裁出 cell → 最近邻放大 8× → images.edit(输入:放大帧+像素风格参考母版;prompt:保持姿势与剪影,替换为新角色外观,限定色板,透明背景);
  2. 结果最近邻缩回原 cell 尺寸 → 量化到 ≤32 色 → alpha 阈值化;
  3. **帧级 QA 硬门**:新帧 alpha bbox 与原帧 IoU ≥ 0.6,越界/漂移帧标记"待重生成",不静默通过;
  4. 全帧就绪后按 atlas 原坐标重组 sheet(尺寸=模板 sheet 尺寸,布局零改动)。
  成本提示:两张 sheet 合计通常 40-80 帧,编辑器在启动前显示帧数与预估调用数,要求确认。
- 两种模式 QA 相同:按 timeline 逐 tick 渲染每个序列的 GIF(帧号→图块规则:`{frame.name}%04d` 缺号沿用前一张,tick 后缀=inclusive 绝对结束 tick——复现 `/pixelart_data` 预览器语义)到 `qa/`,编辑器内逐序列播放。

### S5 voice(语音)

- 对台词表逐条:OpenAI TTS(默认 `gpt-4o-mini-tts`,voice+instructions 从 spec 的声线卡取;同角色全部台词固定同一 voice/instructions)→ 原始输出存 `authoring/` → ffmpeg 转 `44.1kHz 单声道 96kbps CBR -write_xing 0` 存 `build/` → `wf_assets.mp3_encode` 预检(不落 store,只验证能通过);
- ffmpeg 探测:`WF_FFMPEG` env > PATH;缺失时 voice 阶段整体禁用并给出安装指引;
- 编辑器逐条试听/改词重生成/换语速语气重生成。

### S6 qa(校验汇总)

- 汇总:资产清单 vs 模板清单差集必须为空;全部 PNG 可解码且尺寸=模板;MP3 全过严格校验;像素 QA 门全绿;
- 产出 `qa/report.json` + 页面可读版;**报告不绿,apply 按钮不亮**。

### S7 apply(落盘,全在 wf_gui 进程内)

1. `clone_character(src, new_id, new_name, new_code)` dry-run → 用户确认 → 写入(16 表+资产独立);
2. 逐资产 `replace_asset`(自动备份+trim 同步+进 pending;medium/android 前缀自动分包);
3. `save_char_fields` 写身份字段(三层同步);目标属性≠模板 → 调 `element_convert`;
4. 立即 `char_snapshot` 存快照;
5. 输出改动清单;**本阶段任何一步失败即停,指引用快照/备份回滚,不自动继续**。

### S8 publish(发布与发放)

- 复用 `/publish`(pending→三根 diff 包→版本推进→可选重启游戏);
- 发放:提示走服务端 admin 邮件(现有流程),页面给出角色 ID 与操作指引;不新写服务端代码;
- 「导出角色包」按钮:调 `wf_character_pack` 生成 character-pack-v1 manifest 的离线包(为将来接入 `wf_release.py` 事务发布留口,v1 不强制走它)。

### 修改

## 6. 安全与回滚(硬性)

1. S0–S6 **只写 `work/char_gen/`**,不碰 store/cdndata/CDN;
2. apply 全程走既有写端点(自动备份+pending+校验),新代码不得绕过 `replace_asset`/`clone_character` 直写 store;
3. spec 与缓存里**不存 API Key**;日志脱敏;
4. 发布前强制 `asset_template_check` 全绿+QA 报告绿;
5. 回滚三件套照旧:`char_snapshot` zip、`.bak-wfmod-*`、`/char_delete`+反向发布;
6. 不 `git add`/`git commit`(交付为工作区改动,由作者审后自行提交);
7. 不动未跟踪逆向工作区(`decompile/`、`ffdec_26.2.1/`、`弹国服/` 等)与 `web/pages/`、`src/routes/web/`、`web/public/`。

## 7. 测试与验收

- **单测(离线)**:假传输往返(请求构造/重试/缓存命中);派生尺寸=模板实测;recolor 色板映射确定性;repaint QA 门(IoU 拒绝样例);sheet 重组坐标不变;MP3 校验拒绝 VBR fixture;spec 校验与 ID 冲突扫描;GIF 渲染 tick 语义(与 `/pixelart_data` 预览器同一映射函数的黄金样例);
- **`wf_char_gen.py --selftest`**:无 Key 环境可跑,走假传输把全流水线在临时目录演练一遍;
- **GUI 冒烟**:`python mod-tools/wf_gui.py` 启动无异常,新页签可开,无 Key 时降级提示正确;
- **验收清单**:①离线单测全绿;②selftest 通过;③(有 Key 时)真实生成一个角色到 S6 全绿——本条留给作者执行,工具需把"从零到 S6"压缩为 ≤3 次人工确认;④apply/publish 不在实施阶段执行(涉及真实 store,由作者手动触发)。

## 8. 开放问题(实施中遇到按此决策)

- OpenAI TTS 输出格式若无 CBR MP3 选项 → 请求 wav 再 ffmpeg 转码(方案已按此设计,不阻塞);
- gpt-image-1 透明背景:请求 `background: "transparent"` + `output_format: "png"`;若模型忽略,fallback 用色键抠图(容差可调)并在 QA 标注;
- repaint 单帧成本过高时,可选"关键帧重绘+相邻帧色板迁移"的降级档(spec 加 `pixel.repaint_keyframes_only`),但 v1 先做全帧;
- 模板语音清单若含 datamine 缺失的文件 → 以 store 实际存在的为准,清单差集报告在 S0 就显示。
