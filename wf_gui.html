技能级别 移植 / 整技能替换

技能形态切换(条件变身)

新建角色(克隆模板 → 全新 ID)· 实验功能

⚠ 坏档风险:全新 ID 角色属于高危实验,曾多次导致客户端反复闪退与存档损坏 (服务端把存档里"客户端造不出的角色"通告给 /load → 进不去游戏;删除角色后存档里的残留引用同样会崩)。 务必:①创建前先在「角色资料」页给模板角色拍快照;②发放进存档前先在 admin 导出一份存档备份; ③金丝雀验证(列表→详情→编队→战斗)任何一步不对,立即「删除角色」回滚 + admin 从存档移除; ④实验角色不要留在日常使用的存档里。
流程:①填模板与新 ID → 预览 → 创建(写入 ②层 16 张按角色索引的表 + ①层,含写入后校验,自动备份) → ②右上角发布 → ③重启服务端(start-cn.bat) → ④admin 后台把新 ID 加进存档(邮件/发放,跳过扭蛋) → ⑤进游戏金丝雀验证:角色列表→详情页(立绘/玛纳板)→编队→战斗,有问题就下面「删除角色」回滚 + admin 从存档移除。
已覆盖客户端全部 16 张按 character_id 索引的表(角色/词条×6/队长技/文本/数值/觉醒/语音/技能预览/站位/立绘定位/立绘属性/玛纳板/玛纳节点/开板条件/技能升级/抽卡音效)。扭蛋不影响:新角色只是不在任何卡池,用官方邮件/admin 直接发放即可(抽卡音效表已一并克隆,真去抽也不缺)。称号/剧情/任务等进度类表未克隆(不影响显示与战斗,缺了只是没有称号任务)。
资产独立 = 新 code_name + 把模板的立绘/图标/像素/语音/配套数据全套(约 32 个文件)复制到新路径 + action_skill 独立键 —— 改这个新角色不影响模板。不勾 = 与模板共用资产与技能(轻量金丝雀)。

选表 + 填键,点「加载」;或从词条/数值/资料/技能/武器页点「JSON」按钮跳转

改动日志 / 一键回溯

每次写入自动记一条,发布时回填版本号。回溯 = 把该表还原到此条改动之前的状态(该表此后所有改动一并撤销)+ 自动重新发布 + 重启游戏。只想还原不发布,用「备份」页签。

备份 / 还原

「还原」只写回本地表(并加入待发布),不自动发布;要让游戏拉回改动,还原后点右上角发布,或直接用「改动日志」页签的一键回溯。
