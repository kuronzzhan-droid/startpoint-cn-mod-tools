技能级别 移植 / 整技能替换

级别移植:把来源角色某一段技能(名称/描述/能量/动作效果)复制为本角色某段——本角色已有该段=替换,没有=新增(比如给没有＋＋的角色加第 3 段)。整技能替换:来源的全部技能段原样搬来(字节级复制,最稳)。技能动画/效果会变成来源角色的。

技能形态切换(条件变身)

条件满足时技能自动变成另一形态(全库 6 个角色自带:惠惠=有状态时、塔吉斯=HP≥50% 等)。条件存在 character 表 col9-16,切换后技能本体在 switched_action_skill 表——目前只能选已有 6 键(借用=技能变成对方的切换形态)。改「条件/阈值」对自带切换的角色即改即用;条件选「无切换」=清掉。发布后生效。
变体 = 本角色在 switched_action_skill 里引用的切换后技能效果;无变体的角色此处为空

技能 / 词条数值倍率

对本角色(或指定词条 ID)的数值列整体乘倍率——快速全面强化/削弱。作用在词条表上(技能威力字段组等)。

新建角色(克隆模板 → 全新 ID)· 实验功能

⚠ 坏档风险:全新 ID 角色属于高危实验,曾多次导致客户端反复闪退与存档损坏 (服务端把存档里"客户端造不出的角色"通告给 /load → 进不去游戏;删除角色后存档里的残留引用同样会崩)。 务必:①创建前先在「角色资料」页给模板角色拍快照;②发放进存档前先在 admin 导出一份存档备份; ③金丝雀验证(列表→详情→编队→战斗)任何一步不对,立即「删除角色」回滚 + admin 从存档移除; ④实验角色不要留在日常使用的存档里。
流程:①填模板与新 ID → 预览 → 创建(写入 ②层 16 张按角色索引的表 + ①层,含写入后校验,自动备份) → ②右上角发布 → ③重启服务端(start-cn.bat) → ④admin 后台把新 ID 加进存档(邮件/发放,跳过扭蛋) → ⑤进游戏金丝雀验证:角色列表→详情页(立绘/玛纳板)→编队→战斗,有问题就下面「删除角色」回滚 + admin 从存档移除。
