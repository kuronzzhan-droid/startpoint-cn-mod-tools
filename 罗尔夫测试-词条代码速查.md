## 关键代码速查(只列可靠列)

> ⚠️ AMF3 schema 的 enum 内容名**不可靠**(如把 c46=5 误标成 ConditionResistanceYellow;
> 实际 c50=60000 才是「攻击力+60%」)。**数值列(强度/阈值=千分比、帧、能量)才可信**,
> 以面板数值反推语义(字段手册第五章)。

| 效果 | 列/编码 | 说明 |
|---|---|---|
| 技能能量 | action_skill 内层 c4=SLv1 / **c5=满级(面板值)** | 键=code_name;拉平就两列同值 |
| 攻击力% / 强化弹射伤害% | c50/c51(瞬发) · c112/c113(持续) | 千分比,60000=+60% |
| **属性共鸣前置**(雷/火共鸣等) | 前置块 **c7=600000 · c8=600000 · c9=元素**(Red火/Yellow雷…) | 600000=共鸣阈值;删共鸣=清空 c7/c8/c9 |
| **冲刺间隔缩短(Swift)** | 结构:Opening触发 + PF内容(c62/c63) + **帧 c56=18000000(=180帧=3秒)**;枚举名 ConditionSwift=31 / Swift(持续)=420(CN 数据不直接存该值,靠结构定位) | 莱特队长技 131182 **line4** 即此效果 |
| 强化弹射次数上限 / Lv门槛 | c62 power_flip_limit · c63 end_power_flip_limit · c64 Lv门槛 | 11=Lv1以上 |
| 前置块(通用) | c5 precondition · c7~c11 前置1 · c12~c18 前置2 · c19~c25 前置3 | 移植后清前置=去触发条件 |
| 主位限制 | c1 unisonable=false / 任意列=202(OwnerIsMain) | 解除=true / 0 |

## 工具新增能力(本次开发)

- **技能能量编辑**:`wf_mod_tool.py` 加 action_skill 嵌套读写原语(`load/decode/encode/write_action_skill_*`);
  GUI 加 `GET /api/mod/skill_energy?character=` 与 `POST /api/mod/skill_energy/save`。
- **队长技→队长技移植**:`POST /api/mod/copy_leader_to_leader`(整行覆盖,可连描述键)。
- **整词条追加移植**:`POST /api/mod/append_ability`(把来源词条所有行追加到目标,不覆盖原有)。
- **行级移植+清前置**:`POST /api/mod/transplant_line`(单行移植,`strip_cols` 支持 `element_resonance`/`precondition2`/`precondition3` 预设,按目标表类型自动换算列号并把块 kind 回 0;也可传显式列号列表)。
